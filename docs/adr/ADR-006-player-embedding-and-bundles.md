# ADR-006: Player Embedding and Bundle Strategy

**Status:** Accepted (stakeholder sign-off, 2026-07-18)

## Context
FR-13–FR-18, NFR-2, OQ-16 device floor, UX Capability Tiers + Moodle Embedding Contract + Showcase amendment; project-context §2: self-contained embeddable bundle (script + mount API), no Next.js host, no TypeScript.

## Decision
- **One bundle, one global:** Vite 8 lib-mode IIFE exposing `window.EdonPlayer.mount(element, {scriptUrl, token, locale})` / `unmount()` — the canonical mount signature (AD-22); everything else (resume state, flags, governance state, tier hints, attempt ids) arrives via the Player's own `GET /api/v1/playback/bootstrap` call, never via page-embedded data. Advanced overrides (`source`, `sink`, `narration` — ADR-014) are optional and default to platform implementations. Syntax target ES2017 (`build.target: 'chrome61'`, `cssTarget: 'chrome61'`) — verified floor for never-updated Android 8 WebView-shell browsers. Styles are injected, namespaced (`.edon-p-*`), scoped to the mount container; no global resets (Embedding Contract).
- **Runtime:** React 19.2 (constitution §2 holds; Preact rejected for MVP by stakeholder ruling 2026-07-18 — recorded as a future optimisation with a measured ~50 kB gzip core saving). No runtime schema validation; the script arrives server-validated.
- **Chunking (budgets raised 2026-07-18, lazy-chunk discipline retained):** core (shell + slide/narration/quiz/diagram renderers + audio controls + registry/delivery/narration interfaces) ≤ **180 kB gzip, CI hard fail**; lazy chunks fetched from the platform origin relative to the bundle URL: `model3d` (tree-shaken Three.js r185 + DRACOLoader + wasm decoder) ≤ 250 kB gzip, `simulation` runtime ≤ 60 kB, `fonts+motion` (Inter + Plus Jakarta Sans subsets + motion) ≤ 300 kB — **standard**, loaded post-first-paint on all capable devices, never render-blocking. Narration audio assets: ≤ 800 kB per Block, ≤ 10 MB per lesson (in `budgets.json`).
- **Device posture (stakeholder realignment 2026-07-18, supersedes the OQ-16-derived posture):** Showcase/Full is the canonical default — modern smartphones with reliable connectivity assumed. **Auto-load is the default everywhere**; tap-to-load (with transfer-size label) applies only under explicit data-saver signals (`saveData`, `prefers-reduced-data`, metered hints). **Brand webfonts are standard** (post-first-paint, never render-blocking). Tier detection stays feature-based, never user-agent. The Floor/Constrained rungs are retained as thin best-effort fallback (they also serve flag-off states and future offline export) and no longer gate launch. At most one heavy Block live at a time; leaving a heavy Block releases it (HTTP cache makes re-load free). **Constrained predicate** (fallback mechanics): low device class = `deviceMemory ≤ 2` or undefined → per-asset load ceiling halves; any heavy-asset load failure or readiness timeout routes that Block to Constrained behavior.
- **Asset budgets** (in repo-root `budgets.json`, all values bytes, CI + pipeline enforced; raised 2026-07-18): Model3D glTF ≤ 10 MB transfer hard cap (≤ 6 MB preferred at selection time), Draco-compressed via @gltf-transform (MIT at pinned 4.4.x) at library-ingest time; posters ≤ 100 kB, lazy near-viewport; diagram SVG ≤ 200 kB.
- **Old-engine floor is actively enforced**: targeted built-in polyfills ship inside the core budget (`globalThis`, `queueMicrotask`, `Promise.finally`, plus whatever the browserslist compat-lint flags — transpilation does not supply built-ins), no `import.meta.url` in IIFE output (base URL captured from `document.currentScript`), and one real old-engine smoke test is a pre-launch hardening work item (the throttled CI profile runs modern Chromium and cannot catch missing built-ins).
- **Perceived-performance checks** (amended 2026-07-18): the **blocking** CI check runs a standard modern profile (unthrottled Chromium, 4G-shaped network): title + first Slide text ≤ 2.5 s. The low-spec profile (Playwright + CDP `Emulation.setCPUThrottlingRate(6)` + 400 kbps/400 ms RTT + ~1.5 GB memory cap) still runs on every player-affecting change but is **advisory** — it informs the fallback experience, it does not gate merges or launch.
- **Preview parity (FR-9):** the Authoring Preview overlay loads this same bundle and mounts it against the current Draft with a `mode: 'preview'` flag (adds device-width frame + Low-spec toggle that forces the Floor tier); no separate preview renderer exists.

## Consequences
- The Moodle plugin's embed is three lines (div + script + mount) — thin by construction (WI-MOD-1).
- SCORM/offline export (V3 seam e) reuses the same IIFE against a `file://`-safe `CompleteScriptSource`.
- The budgets file is the single tuning point; changing a budget is config + CI, not archaeology.
